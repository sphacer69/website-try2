Choose how customers pay after confirming a booking. Dynamic Checkout charges the exact total based on what they ordered (recommended). Static Payment Link always charges the same amount you set in Stripe.


            

            

            


            

              

              
Create one in your Stripe dashboard → Payment Links. The amount is whatever you set in Stripe.

            


            

              

              

              

              

              

                ▸ How to set up the Checkout Endpoint
                

                Dynamic Checkout needs a tiny serverless function (≈50 lines) that holds your Stripe secret key and creates a Checkout Session per order. Your secret key NEVER goes in this admin panel — it lives on the server.


                Easiest path: Cloudflare Workers (free, no credit card).
                
    
                  
  1. Sign up at cloudflare.com/workers
    2. 
                  
  2. Create a Worker → paste the worker code (download from your project files: stripe-worker.js)
    3. 
                  
  3. Settings → Variables → add STRIPE_SECRET_KEY (your sk_live_… key) as an encrypted variable
    4. 
                  
  4. Variables → add ALLOWED_ORIGIN = your website URL (e.g. https://yourdomain.com)
    5. 
                  
  5. Deploy. Copy the Worker URL into Checkout Endpoint URL above.
    6.

Customer Confirmation Email

              
When a customer books, also send them a confirmation email. The same webhook is used — you just add a 2nd "Send Email" action to your Zap and map it to the customer's address. Setup steps below ↓

              

              

              

              

              

              
Available placeholders: {customer_name} {customer_email} {customer_phone} {vehicle} {service} {date} {time} {final_price} {business_name} {business_phone} {business_email} {business_address}

              

                
                
              

              

                ▸ How to add the customer confirmation step in Zapier
                

                  
    
                    
  1. Open your existing Zap (the one already sending you admin notifications)
    2. 
                    
  2. Click the + button below your last action to add a new step
    3. 
                    
  3. Pick Email by Zapier → Send Outbound Email (or Gmail / Outlook for nicer formatting)
    4. 
                    
  4. Configure the fields:
                      
      
                        
    • To: map the customer_email field from the trigger
      • 
                        
    • Subject: map confirmation_subject
      • 
                        
    • Body: map confirmation_body
      • 
                        
    • From Name: map confirmation_from_name
      • 
                        
    • Reply To: map confirmation_reply_to
      • 
                      
    
                    
    5. 
                    
  5. (Optional, recommended) Add a Filter by Zapier step before this email action with the rule "send_confirmation equals 1" — so customer emails only go out when you have the toggle above turned on.
    6. 
                    
  6. Re-publish your Zap. Test it with the "Send Test Email" button above.
    7. 
                  

                  
The confirmation email body is rendered with the placeholders filled in on our side — Zapier just sees the final text. So you don't need to map individual fields like {customer_name} in Zapier; just map confirmation_body and you're done.
